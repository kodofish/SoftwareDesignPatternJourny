下載 Astah Professional License
首先到這個網址: https://astah.net/downloads/
尋找以下產品 Astah Professional，按下 Download 按鈕。（它會要求你填一點點資料）
下載 Astah Professional 之後，再來你需要 License 才能夠解鎖 Astah Profession 的使用資格，請遵照以下步驟執行。
第一步：設置環境變數 RLM_PROJECT
首先，你必須設置「啟動 Astah Professional 時的環境變數」： 設置環境變數 RLM_PROJECT=wsa-dpo01-2022
不同作業系統的環境變數設法都不同，請選擇自己所使用的作業系統部分觀看：
Mac
按下 cmd + 空白，輸入 terminal 來開啟終端機。
在終端機中送出指令 launchctl setenv RLM_PROJECT wsa-dpo01-2022
如此一來，在開啟桌面應用程式 Astah Professional 時，就會帶著此環境變數。
檢查 Astah 真的有吃到環境變數：
先透過 Mac 上面的 Activity Monitor 或是使用 ps -ax | grep astah 指令查詢 Astah Pro 的 process ID。接著使用 ps eww <Process ID> 指令就能夠看到 Astah Pro 實際在執行時吃到的環境變數囉。
Windows
前往控制台 > 系統及安全性 > 系統 > 進階系統設定
點選「環境變數」
點選上面使用者變數面板下方的「新增」 （注意，必須新增使用者變數，而非新增下方的系統變數）
輸入變數名稱 RLM_PROJECT以及變數值 wsa-dpo01-2022（如圖） 輸入完之後按下確定，查看變數是否新增成功。
確認成功後，按下確定，這樣就設置完環境變數了。
第二步：打開 Astah Pro 並設置 License
打開剛下載好的 Astah Professional，打開後預設有七天期限的試用期。但我們都是正式旅程的冒險者，是「用好用滿的啦！」：

小複習
屬性 (Attribute) 語法為 <權限符號> <屬性名稱>：<屬性類型> - id: int 代表一個私有的 Integer 屬性名為 id。
權限符號主要有以下三種：
公開 Public (+)：任何類別皆可存取
保護 Protected (#)：只有自身類別內部，以及自己的子類別能夠存取。（有些程式語言的保護權限還添加了允許同 package 下的類別也能存取，如：Java）
私有 Private (-)：只有自身類別內部能存取。
動作 (Operation) 語法為 <權限符號> <動作名稱>：<動作回傳值類型> 程式語言中的方法 (Method) 或函數 (Function) 即為 動作 (Operation) 的實作。
其他種類的關聯
如果你想要拉的是「聚合」或是「複合」，可以點選關聯符號右側的小箭頭，就能展開所有關聯類型囉！

將類別轉為介面 (Interface)
你可以在任一類別上點擊右鍵，選取 ‘Convert to Interface’
然後這個類別就會變成 Interface，類別名稱的頭上會多出 <<interface>>
注意這個 << stereotype >> 語法，在 UML 中我們會使用這種被稱之為 Stereotype (刻板型態，原意為刻板印象) 的語法來在一些建模符號上說明其用途 (Purpose)。
